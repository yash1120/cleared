# Deploying Cleared

Three ways to show Cleared to people, fastest first. All of them work **with no API key**
(offline mock mode) and seed a polished demo dataset automatically, so you always land on a
populated dashboard.

> **Demo login (seeded everywhere):** `demo@cleared.com.au` / `demo1234` — this account is also
> an admin, so it can see the Admin console too.

---

## Option A — Run locally for a screen-share (fastest)

**Windows (PowerShell):**
```powershell
cd F:\projects\cleared
.\run.ps1
```

**macOS / Linux:**
```bash
cd cleared
./run.sh
```

The script creates the venv, installs deps, builds the frontend if needed, seeds demo data, and
launches on <http://localhost:8000>. Flags: `-Build` / `BUILD=1` to force a frontend rebuild,
`-Fresh` / `FRESH=1` for a clean database, `-Port 9000` / `PORT=9000` to change the port.

To demo **live Claude** instead of mock mode, set your key first:
```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."   # PowerShell
export ANTHROPIC_API_KEY=sk-ant-...      # bash
```

## Option B — A public live URL (Render, free)

Best when you want to send someone a link.

1. Push this repo to GitHub.
2. In [Render](https://render.com) → **New → Blueprint** → pick the repo. Render reads
   [`render.yaml`](./render.yaml) and provisions a Docker web service.
3. Click **Apply**. First build takes ~3–5 min (it builds the frontend, then the Python image).
4. Open the `*.onrender.com` URL → log in with the demo credentials above.

Notes:
- The free plan filesystem is **ephemeral** and the service **sleeps after 15 min idle**. Cleared
  re-seeds demo data on every cold boot (`CLEARED_SEED_DEMO=1`), so the link is never empty —
  the first request after sleep takes a few seconds to wake.
- `CLEARED_SECRET_KEY` is auto-generated by Render and kept stable.
- To use live Claude, add `ANTHROPIC_API_KEY` in the Render dashboard (Environment tab).
- For durable storage, move to a paid plan and uncomment the `disk` block in `render.yaml`.

## Option C — Fly.io (Sydney region, persistent volume)

Best for the AU data-residency story and a DB that survives restarts.

```bash
fly launch --no-deploy --copy-config --name cleared-demo
fly volumes create cleared_data --region syd --size 1
fly secrets set CLEARED_SECRET_KEY=$(openssl rand -hex 32)
fly secrets set ANTHROPIC_API_KEY=sk-ant-...   # optional — omit for mock mode
fly deploy
fly open
```

[`fly.toml`](./fly.toml) pins `primary_region = "syd"`, mounts the volume at `/data`, health-checks
`/api/health`, and scales to zero when idle.

---

## Option D — Plain Docker (any host)

```bash
docker build -t cleared .
docker run -d --name cleared -p 8000:8000 \
  -v cleared-data:/data \
  -e CLEARED_SECRET_KEY=$(openssl rand -hex 32) \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  cleared
# open http://localhost:8000
```

The image is multi-stage: stage 1 builds the React frontend with Node, stage 2 is a slim Python
runtime that serves the API + the built SPA. The entrypoint
([`docker-entrypoint.sh`](./docker-entrypoint.sh)) seeds demo data when the DB is empty, then
launches uvicorn on `$PORT` (default 8000).

## What "live" vs "mock" means

| | Mock mode (default, no key) | Live mode (`ANTHROPIC_API_KEY` set) |
|---|---|---|
| Risk assessments | Hand-authored + heuristic, instant, free | Generated by Claude Sonnet 4.6 |
| Citation verification | ✅ same verifier | ✅ same verifier |
| Everything else (auth, records, audit, charts, admin, PDF) | ✅ identical | ✅ identical |

Mock mode is genuinely useful for a demo: it's instant, costs nothing, and exercises the entire
product surface. Flip to live only when you want to show real model-generated assessments.

---

## Going to production

Showing a demo is not the same as running real customer data. Before that, work through
[`GO-LIVE.md`](./GO-LIVE.md): rotate the secret key, have a lawyer review the rule pack, enrol with
AUSTRAC, wire OpenSanctions + an ID-verification provider, configure SMTP, host in an AU region,
and add the auth hardening (refresh tokens, password reset, RBAC). **Turn off `CLEARED_SEED_DEMO`
and never ship the `demo@cleared.com.au` account to production.**
