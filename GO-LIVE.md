# Cleared — go-live checklist

Cleared is a working application — auth, multi-profession CDD/SMR, sanctions screening,
integrations (Xero/MYOB/CSV/API keys), signed outbound webhooks, ongoing-monitoring with a
reviews schedule, an immutable audit trail, dashboard analytics, customer history chains,
review-reminder digests, and an operator admin console. The items below are what only you (or
a specialist) need to bring before running it for real customers.

> Cleared is software, not legal advice. It does not, by itself, make a firm AML-compliant.

## 1. Intelligence
- [ ] **Anthropic API key** — set `ANTHROPIC_API_KEY`. Without it the app runs in **mock mode**
      (canned/heuristic output). With it, assessments are generated by Claude (Sonnet 4.6 by default).

## 2. Legal & regulatory (blocking for production)
- [ ] **Have an AML/CTF lawyer review and complete the rule pack**
      (`cleared/rules/austrac_real_estate_rules.json`). It is a paraphrased MVP subset, not the law.
- [ ] The firm must **enrol with AUSTRAC** and adopt its own AML/CTF program — the reporting
      entity's obligation, not the software's.
- [ ] **Privacy**: address the Privacy Act 1988 / APPs, publish a privacy policy (see `/privacy`),
      and sign a data-processing agreement with each customer.

## 3. Real data providers (swap the MVP stand-ins)
- [ ] **Sanctions/PEP screening**: point at OpenSanctions — set `CLEARED_OPENSANCTIONS_URL`
      (and `CLEARED_OPENSANCTIONS_KEY` for the hosted API) or self-hosted *yente*.
- [ ] **Identity verification**: integrate a provider (FrankieOne, GBG, Sumsub) for
      document/biometric verification — the demo does not verify IDs.

## 4. Hosting & data residency
- [ ] Host in an **Australian region** (ap-southeast-2) with HTTPS termination.
- [ ] Persist `CLEARED_DB_PATH` to durable, backed-up storage. For scale, swap in managed Postgres
      by reimplementing `cleared/store.py` (everything funnels through that module).
- [ ] Backups + enforced 7-year retention.

## 5. Access & accounts
- [x] **Authentication** — email/password (pbkdf2) + JWT bearer; **set a strong `CLEARED_SECRET_KEY`**
      in production (tokens are signed with it).
- [x] **API keys** (machine-to-machine) with SHA-256-only storage and per-user scope.
- [x] **Per-user record isolation** enforced server-side on every query.
- [x] **Admin console** — set `CLEARED_ADMIN_EMAILS=a@you.com,b@you.com`; bootstrap via
      `python -m cleared.cli seed-admin a@you.com`.
- [ ] Harden further: httpOnly-cookie tokens, refresh tokens, email verification, password
      reset, rate limiting, per-firm team accounts with RBAC.

## 6. Email (review-reminder digests)
- [x] Reminders ship out of the box. Configure SMTP via `CLEARED_SMTP_HOST` / `_PORT` / `_USER` /
      `_PASSWORD` / `_FROM`. Without them, `/reminders/send` returns `mock: True`.
- [ ] Schedule the daily job (cron / systemd timer / GitHub Actions):
      ```
      0 8 * * *  cd /opt/cleared && python -m cleared.cli run-reminders
      ```

## 7. Outbound webhooks
- [x] HMAC-SHA256 signed deliveries; signing secret shown once at creation.
- [ ] Document the verification snippet for partners (header `X-Cleared-Signature`).

## 8. Commercial
- [ ] Billing (Stripe). The `/pricing` page currently shows "Contact" tiers.
- [ ] Terms of service (draft at `/terms`), support contact, SLAs.

## Run modes

### Local / pilot (mock unless `ANTHROPIC_API_KEY` is set)
```bash
cd cleared
python -m uvicorn cleared.api:app --reload --port 8000
# In another shell, build the frontend (one-off):
cd frontend && npm install && npm run build
```

### Docker
```bash
docker build -t cleared .
docker run -p 8000:8000 \
  -v cleared-data:/data \
  -e CLEARED_SECRET_KEY=$(openssl rand -hex 32) \
  -e CLEARED_ADMIN_EMAILS=you@yourdomain.com \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  cleared
```

### Operator commands
```bash
python -m cleared.cli seed-admin you@yourdomain.com   # interactive password prompt
python -m cleared.cli info                             # snapshot of users/records/signups
python -m cleared.cli run-reminders                    # build + send overdue-review digests
python -m cleared.cli export-signups signups.csv       # dump the beta inbox
```
